# OfferFree 全方位优化与实施说明

## 本次已完成

- 全新响应式视觉系统：使用用户指定的卡通头像作为品牌与浏览器图标，重做首屏、岗位卡片、标签、截止倒计时、抽屉详情、移动端底部导航和加载骨架。
- 招聘信息库：从 `daily-jobs.json` 读取公开岗位，自动去重，并过滤“应届生求职网”“中智招聘网”等平台占位条目，平台只作为来源展示。
- 八类来源框架：应届生求职网、中智招聘网、24365、国家公共招聘网、国资委招聘、企业招聘官网、高校就业网、智联招聘。
- 筛选与排序：全国主要城市、行业/专业、学历（含专升本和博士）、届别、岗位类型、企业属性、来源，以及最新/截止/匹配度/薪资排序。
- 岗位专区：27 届提前批、26 届补录、暑期实习、国企央企。
- 岗位详情：JD 摘要、专业与学历要求、来源、截止时间、风险提示、同公司岗位、收藏和加入投递。
- 求职工作台：收藏库、待投递、已投递、笔试、面试复盘、拒绝、意向 Offer、已拿 Offer；支持详细记录和 CSV 导出。
- 附件简历：取消文字简历，仅保留 PDF/Word 多版本附件库；文件保存在浏览器 IndexedDB。
- JD 匹配：根据学历、专业/行业、目标城市、届别和技能关键词在本地计算匹配分。
- 待办：自动生成即将截止和未完成面试复盘待办，并支持自定义待办。
- 校友去向入口：只展示有来源、已匿名化的统计；无数据时明确提示，不伪造校友结果。
- 私密反馈：取消 GitHub 公共建议箱，改为调用用户邮件客户端，收件人为 `tjiale49@gmail.com`。
- 无登录：按最新要求取消登录与账号入口，打开网站即可使用。

## 数据采集与合规边界

当前自动更新脚本扩展至八类公开来源，并保留来源健康状态。若网站存在登录墙、验证码、robots 限制或禁止自动抓取，脚本不会绕过保护；应改用官方 API、RSS、站长授权数据或人工审核。

OfferBiu 全量岗位不能在未获许可时直接复制。合规方案按优先级为：

1. 获得 OfferBiu 书面授权与 API/数据导出；
2. 由站长提供有权使用的 CSV/JSON 定期导入；
3. 仅保存指向原站的链接与必要摘要，不复制受版权保护的完整内容。

## 下一阶段建议

### P0：数据可靠性

- 为每个平台开发独立解析器，不使用一个通用 HTML 规则覆盖全部网站。
- 增加 URL 状态检测、失效标灰、人工复核队列和岗位过期归档。
- 统一城市、学历、专业、企业性质词典，建立“原始值 → 标准值”映射。
- 为企业官网建立可维护的白名单，不尝试无边界抓取整个互联网。

### P1：求职闭环

- 投递看板增加拖拽、批量修改、批量导出、日历 `.ics` 导出和浏览器通知。
- JD 匹配增加附件简历文本解析；解析前必须明确征得用户同意，并默认只在本地处理。
- 面试复盘生成个人统计图表：投递趋势、面试率、Offer 率、城市和行业分布。

### P2：校友数据

- 数据源仅使用高校就业质量报告、公开年度报告和用户自愿匿名贡献。
- 设置最小样本量，避免通过小样本反推个人去向。
- 每条统计展示学校、年份、报告页码或公开链接。

### P3：真正的私密反馈

当前 `mailto:` 需要用户在邮件客户端确认发送。若要网页内直接发送，需要部署后端邮件接口，并配置验证码、频率限制、垃圾信息防护和密钥管理；邮箱地址和服务密钥不能直接写进前端代码。

## 发布顺序

1. 先发布本次静态改版，确认 GitHub Pages 页面、岗位加载、筛选、移动端和本地工作台正常。
2. 手动运行一次 Daily recruitment update，检查八类来源的更新状态与误抓取情况。
3. 对误抓取来源暂停自动发布，完善独立解析器后再开启。
4. 获得 OfferBiu 数据授权后再开发迁移适配器。
